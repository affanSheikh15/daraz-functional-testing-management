# Jira-Bug-Reports
Manual testing and bug tracking project using Jira, covering defect life cycle, issue reporting, and QA workflow simulation.

## Overview
This project demonstrates a complete Manual Testing and Quality Assurance workflow using Jira.
It covers all stages of the Software Testing Life Cycle (STLC), from identifying defects to reporting, prioritizing, and tracking them until resolution.
The purpose of this project is to simulate real-world defect management and showcase practical QA skills such as test case writing, bug reporting, and issue tracking.

## Key Learning Areas
* Software Testing Life Cycle (STLC)
* Defect Life Cycle (Bug Life Cycle)
* Manual Testing Workflow
* Bug Reporting and Tracking
* Test Case Design and Execution
* QA Documentation and Prioritization

## Tools Used
* Jira – Bug tracking and defect management
* TestRail / Excel – Test case writing and tracking (optional reference)
* VS Code – Documentation workspace
* GitHub – Project hosting and version control

## Bug Reports Included
Each issue contains:
* Bug Title
* Steps to Reproduce
* Expected Result
* Actual Result
* Environment Details
* Priority and Severity
* Status Tracking

## Example:
Bug ID: QPP-1
Title: Login allows blank password

## Steps to Reproduce:
1. Go to the login page
2. Leave the password field blank
3. Click the "Login" button

Expected Result: System should display an error message “Password is required.”
Actual Result: System logs in without validation.
Priority: High
Status: Open

To gain practical experience in manual QA processes using Jira and demonstrate a solid understanding of how QA engineers document, prioritize, and track defects in a real testing environment.
